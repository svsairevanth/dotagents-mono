# Jobs To Be Done

## Core job
When I hand work to DotAgents, help me move that work forward with enough transparency and control that I can trust the outcome.

## Functional jobs

### Start work quickly
When I know what I want, I want to launch a task without having to compose a perfect prompt.

### Understand current state instantly
When a task is running, I want to know what is happening now without reading the whole history.

### Inspect execution when needed
When something looks wrong or risky, I want to inspect the actions taken and the reasoning behind them through observable evidence.

### Intervene safely
When I need to redirect the system, I want to do it without breaking momentum.

### Review outputs clearly
When the system says it is done, I want to evaluate artifacts, diffs, drafts, and verification before I accept the result.

### Resume context fast
When I return later, I want to understand what happened, what remains, and where I should focus next.

## Emotional jobs

- Help me feel in control even during autonomous execution
- Help me feel confidence without needing blind faith
- Help me feel momentum instead of ambiguity
- Help me feel calm rather than overloaded

## Social jobs

- Help me trust using agents in front of collaborators or teammates
- Help me justify decisions based on visible execution and outputs

## Anti-jobs
The redesign should avoid creating these burdens:

- forcing users to scroll through long chat logs to understand status
- making users guess whether the agent is thinking, stuck, or done
- burying risk behind polished language
- making intervention feel like starting over
